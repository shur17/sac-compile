术语表：
1. 业务集群：业务使用的 SequoiaDB 集群（MySQL实例组+存储集群）
2. 业务主机：业务集群所在的主机（1台 ~ N台）
3. SAC Agent：SAC Agent 是 SAC 的采集器，部署在业务主机上，一台主机一个 SAC Agent
4. SAC 主机：SAC 需要部署在独立一台主机上，在文档都叫 SAC 主机
5. SAC 数据库：SAC 数据库是 SequoiaDB 存储集群，1个数据组1个数据节点，部署在 SAC 主机上，用来存储 SAC 的数据
6. SAC Agent Proxy：SAC Agent Proxy 是 SAC 的采集服务端，部署在 SAC 主机上，用于接收 SAC Agent 采集回来的数据


一、环境要求
  1. 业务集群环境要求
     1.1 业务集群要有 SQL 实例组
    
     1.2 存储集群的节点配置要把 mongroupmask 设置为 "slowQuery:detail"，该配置项可以给每个 SDB 节点开启慢查询监控。

     1.3 需要根据业务情况适当调大 MySQL 配置项 performance-schema-max-sql-text-length（单位 bytes，默认值为 1024 ），
         该配置项可以指定 SQL_TEXT 的最大长度，SQL 查询语句长度大于 performance-schema-max-sql-text-length 的部分将会被截断。【不是必须修改】

     1.4 需要根据业务情况适当调大 MySQL 配置项 performance-schema-max-digest-length（单位 bytes，默认值为 1024 ），
         该配置项可以指定 DIGEST 的最大长度，SQL 查询摘要长度大于 performance-schema-max-digest-length 的部分将会被截断。【不是必须修改】

  2. SAC 环境要求
     2.1 SAC 主机要开放端口号：28000 和 28081，其中 28000 是 SAC 的网页服务端口号，28081 是 SAC Agent Proxy 向 SAC Agent 开放的 TCP 连接默认端口号
    
     2.2 SAC 主机与安装 Agent 的主机的时区需要一致，时间误差不超过 10 秒
    
     2.3 SAC 主机至少需要 12 GB 内存
    
     2.4 SAC 主机要安装 Java 环境（Java 版本在 Java8 ~ Java18 之间）


二、整体流程说明
  1. 在 SAC 主机解压 SAC 安装包： tar -zxvf sac-<version>-release.tar.gz

  2. 在业务主机安装 SAC Agent：将 SAC 安装目录下的 agent 目录拷贝到业务主机上（每一台业务主机都要有 agent ），安装启动 SAC Agent 的具体步骤见 agent/bin/Readme

  3. 搭建 SAC 数据库：在 SAC 主机上部署一个 SequoiaDB 集群（1个数据组，1个数据节点），用来给 SAC 存储数据。

  4. 部署 SAC 服务：具体步骤见 bin/Readme

  5. 在 SAC 添加已有的业务集群：
     访问 SAC 网页（访问地址为http://$ip:$port"， 其中 $ip 为 SAC 主机 IP，$port 为 SAC 的网页服务端口号（默认为28000），
     默认账号为 admin，默认密码为 Admin@1024，依次点击“集群管理” -> “发现集群”，输入集群名、协调节点地址、存储集群鉴权、MySQL实例组鉴权，点击“确定”。


三、注意事项
  1. MySQL 实例必须是在 SQL 实例组里，才可以被采集到信息
  2. 检查业务主机和 SAC 主机时间，可以用命令 date 来确认；要确保业务主机和 SAC 主机的时间不超过 10 秒
  3. 检查业务主机和 SAC 主机时区，可以用命令 cat /etc/timezone 来确认是否一致
  4. SAC Agent 配置文件 agent/conf/application.yml 的配置项 hostId 在不同业务主机下必须唯一，且需要保证配置项 SAC Agent Proxy 连接地址 apAddresses 是 SAC 主机的地址
  5. SAC 配置服务配置文件 conf/sac-config/properties/application-common-prod.yml 需要保证配置项 SequoiaDB 连接地址 addresses 有效（SAC 数据库默认都要求安装在 SAC 主机上，这个配置就不需要修改）
  6. 修改系统时区和系统时间时，要先停止 MySQL 实例，不然会引起 MySQL 实例时间 uptime 异常
